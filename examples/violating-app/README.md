# violating-app

A minimal Node service that **deliberately violates** the org engineering standards. It is the
**known-bad fixture** for `standards-review` — it should produce multiple **BLOCK** findings plus
warnings and required-but-unenforceable notes.

Planted violations (each marked with a `!! DELIBERATE VIOLATION !!` comment in the source):

| Where | Rule | Severity · point | Expected |
|---|---|---|---|
| `src/config.mjs` | `sec-no-plaintext-secrets` | Policy · ci-gate | **BLOCK** (+ cite references) |
| `src/db.mjs` | `arch-no-local-sql-databases` | Strategic · ci-gate | **BLOCK** |
| `test/server.test.mjs` | `qual-no-skipped-tests` | Strategic · ci-gate | **BLOCK** |
| `src/server.mjs` | `obs-no-pii-in-logs` | Strategic · audit | REQUIRED (unenforceable) |
| `src/server.mjs` | `obs-structured-logging` | Handbook · audit | ADVISORY |
| `.github/workflows/ci.yml` | `integ-pin-third-party-actions` | Handbook · ci-gate | **WARN** |

The fake secret in `config.mjs` is an obviously-marked placeholder, not a real credential.

The `standards-review` skill / `standards-enforcer` agent under `.claude/` are generated by the
Claude Code adapter (`node adapters/claude-code/build.mjs --out examples/violating-app`).
