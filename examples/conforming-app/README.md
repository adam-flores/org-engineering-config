# conforming-app

A minimal, dependency-free Node service used as the **known-good baseline** for the org engineering
standards. It is written to satisfy the rules so that `standards-review` reports **zero BLOCK
findings** against it.

What it demonstrates:

- **No plaintext secrets** — `API_TOKEN` comes from the environment; `.env` is gitignored; only
  `.env.example` is committed.
- **Real, non-skipped, deterministic tests** — `node --test` in `test/`.
- **Structured logging, no PII** — JSON logs with consistent fields and a correlation id, never user data.
- **Health endpoint** and a pure, unit-testable handler.
- **Pinned CI action by digest** in `.github/workflows/ci.yml`.

Run:

```
node --test        # tests pass
node src/server.mjs # starts on PORT (default 3000)
```

The `standards-review` skill / `standards-enforcer` agent under `.claude/` are generated by the
Claude Code adapter (`node adapters/claude-code/build.mjs --out examples/conforming-app`).
